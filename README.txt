Описание используемых данных: в качестве источника данных выступал портал "Жалобы Москвы" (https://gorod.mos.ru/), где жители города оставляют свои обращения. 

Описание выбранной методологии обработки данных: на сайте уже есть интерактивная карта, где можно посмотреть все оставленные обращения, но эта карта никак не предоставляет аналитику по районам города, тяжело понять кластеризацию жалоб определенной категории, либо выявить наиболее проблемные зоны. Первоначально данные были спаршены с сайта, всего было получено более 3000 обращений, затем из обращения заявителя вычленены адрес, краткая выжимка обращения и определена категория. Полученные данные агрегированы в следующие виды визуализации: слой со всеми обращениями, которые категоризованы по цветам для понимания общего распределения, слой с полигонами районов Москвы, в пределах каждого была определена превалирующая категория жалоб. Однако при определении превалирующей категории была исключена категория ЖКХ и коммунального хозяйства, так как она лидирует в 72% районов и это не дало бы представление о распределении остальных категорий. Также отдельно по каждой категории можно посмотреть распределение обращений, которые агрегированы в сетку (grid) и кластеризованные точки. По каждому району можно посмотреть суммарное количество обращений, но для визуализации полученные результаты нормированы и для определения откуда поступает наибольшее количество жалоб, данные скорректированы на численность населения: определено количество жалоб на 20000 человек по каждому району.

Что было самым сложным в работе: парсинг и обработка полученных данных, несмотря на использование ии при обработке, все равно приходится перепроверять каждую строку таблицы на корректность. 

Что получилось в работе и вы можете этим гордиться: получилось все задуманное и итоговый продукт может быть интересен, например, органам власти для работы с обращениями жителей. В период предстоящих выборов (ГД - 2026 год, Муниципальные выборы - 2027 год) карта полезна для работы с потенциальными избирателями. 

Что не получилось и почему, что можно было бы добавить в гипотетической версии 2: карту можно дорабатывать в качестве CRM: добавить функцию добавления обращений, добавить возможность отмечать статус обращения (в работе / проблема решена).
